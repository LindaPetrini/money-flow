## Overview

Enriched the CSV import flow with an interactive 2-call AI session: uncertain transaction Q&A with merchant memory, followed by recurring expense detection with pre-fill UX. Added quality-of-life improvements: FOUC-free dark mode, optional invoice source field, and 5-filter history search.

## Milestone Summary

**Key Decisions:**
- Merchant pre-classification runs synchronously before API call (cheaper, no latency)
- 2-call AI session design: one combined call (buckets + uncertain transactions), one floor detection call with Q&A context — at most 2 calls per CSV import regardless of file size
- `pendingFloorItem` lifted to `SettingsPage` (not `CsvAiSection`) to enable cross-section pre-fill
- FOUC prevention via inline `<head>` script reading `mf_theme` before CSS paint

**Issues Resolved:**
- Tailwind v4 dark variant selector requires `dark` class on root element, not `:dark`
- `callAnthropicAPI` deprecated (not deleted) to preserve v1.0 backward compatibility

**Issues Deferred:**
- Merchant→bucket mapping management UI (view/edit saved mappings) — AIAN-EXT-01
- History CSV export — AIAN-EXT-02

**Technical Debt Incurred:**
- `callAnthropicAPI` still present but deprecated — can be removed in v1.2
