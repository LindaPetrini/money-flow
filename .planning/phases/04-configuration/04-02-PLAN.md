**State design:**
- `editingId: string | null` — which account row is in edit mode (null = none)
- `draft: Partial<Account>` — local edits for the row in edit mode
- `showAddForm: boolean` — whether the "Add account" inline form is visible
- `newAccount: { name: string; role: AccountRole; targetCents: string }` — controlled inputs for new account form

**Layout — Account list:**
Each account renders as a row with two modes:
- **View mode** (editingId !== account.id): Show `account.name`, role badge, target balance (`targetCents === 0 ? 'No target' : formatCents(account.targetCents as Cents)`), then "Edit" and "Delete" buttons.
- **Edit mode** (editingId === account.id): Show inputs for name (text), role (select), and target (text showing euros e.g. "1200.00"). Two buttons: "Save" and "Cancel".

**Role options for the select element:**
```
income-hub | spending | savings | tax | investing
```

**Edit save logic:**
```typescript
const handleSaveEdit = async () => {
  const updated = accounts.map(a =>
    a.id === editingId
      ? {
          ...a,
          name: draft.name ?? a.name,
          role: draft.role ?? a.role,
          targetCents: draft.targetStr !== undefined
            ? (parseCents(draft.targetStr) as number)
            : a.targetCents,
        }
      : a
  );
  await setAccounts(updated);
  setEditingId(null);
  setDraft({});
};
```

Use a separate `targetStr` string field in draft for the text input value, converting with `parseCents` on save. Do NOT store raw floats in targetCents mid-edit.

**Delete logic:**
Use `window.confirm('Delete account "${account.name}"? This cannot be undone.')`. If confirmed, filter account out and call `setAccounts(filtered)`.

**Add account form:**
Shown below the list when `showAddForm` is true. Fields: name (text, required), role (select), target (text, euros). "Add" button calls:
```typescript
const newAcc: Account = {
  id: crypto.randomUUID(),
  name: newAccount.name.trim(),
  role: newAccount.role,
  balanceCents: 0,
  targetCents: newAccount.target ? (parseCents(newAccount.target) as number) : 0,
};
await setAccounts([...accounts, newAcc]);
setShowAddForm(false);
setNewAccount({ name: '', role: 'spending', target: '' });
```

An "Add account" button at the bottom of the list toggles `showAddForm`. "Cancel" in the add form hides it without saving.
