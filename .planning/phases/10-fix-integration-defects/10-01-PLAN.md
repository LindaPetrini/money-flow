# Plan 10-01: Fix ALLOC-02 — Floor Item Coverage Marking

## Goal

Fix the integration defect where multiple floor items sharing the same destination account are incorrectly all marked as covered when only one was funded. The root cause is `handleDone()` using `destinationAccountId` to identify which floor items were covered — but multiple floor items can share the same account. The fix adds `floorItemId?: string` to `AllocationMove` so the engine tags each floor move with the specific floor item ID, and `handleDone()` uses that ID for precise matching.

## Context

**Bug location chain:**
1. `src/types/domain.ts:36` — `AllocationMove` interface — needs `floorItemId?: string` added
2. `src/domain/allocationEngine.ts:106` — `stabilize()` pushes floor moves without `floorItemId`
3. `src/features/invoice/InvoicePage.tsx:75-103` — `handleDone()` collects `destinationAccountId` from floor moves and matches `floorItems` on `f.destinationAccountId` — BOTH are wrong

**Correct approach:**
- `AllocationMove` gains `floorItemId?: string` (optional — distribute/tax moves don't have a floor item)
- `stabilize()` sets `floorItemId: floor.id` on each move it generates
- `handleDone()` step 2 collects `move.floorItemId` (filtering undefined), step 5 matches `f.id === floorItemId`

**Test impact:**
- Existing tests in `allocationEngine.test.ts` access `move.destinationAccountId` and `move.rule` — adding an optional field is non-breaking, no test changes needed for existing tests
- Add one new test: stabilize moves include `floorItemId` matching the floor item's `id`
- The `AllocationResult structure` test at line 309-322 checks `destinationAccountId` is truthy — this still passes since floor moves still have `destinationAccountId` (the destination account hasn't changed, only the coverage discriminator has)

```typescript
// ─── ALLOC-02 fix: floorItemId on stabilize moves ────────────────────────────

describe('ALLOC-02 fix: stabilize moves carry floorItemId', () => {
  it('each floor move has floorItemId matching the floor item id', () => {
    const floor1 = makeFloor('rent', 120000, 1);
    const floor2 = makeFloor('utilities', 20000, 2);
    const settings = baseSettings({
      floorItems: [floor1, floor2],
    });
    const accounts = [emptyBuffer, taxAccount];
    const result = computeAllocation(300000 as Cents, accounts, settings, TODAY);
    const floorMoves = result.moves.filter(m => m.rule === 'floor');
    expect(floorMoves).toHaveLength(2);
    expect(floorMoves[0].floorItemId).toBe('rent');
    expect(floorMoves[1].floorItemId).toBe('utilities');
  });

  it('tax and distribute moves do not have floorItemId', () => {
    const accounts = [fundedBuffer, taxAccount];
    const result = computeAllocation(200000 as Cents, accounts, baseSettings(), TODAY);
    const nonFloorMoves = result.moves.filter(m => m.rule !== 'floor');
    for (const move of nonFloorMoves) {
      expect(move.floorItemId).toBeUndefined();
    }
  });
});
```
