<objective>
Create the `StorageErrorContext` bridge and add `NotAllowedError` try/catch guards to all three store write paths.

Purpose: Eliminate silent data loss when FSA permission is revoked mid-session. Stores catch `NotAllowedError` on write, call `reportPermissionLost()` to signal the React UI, and return without re-throwing (memory state is already updated). The React provider wires the module-level setter after mount so non-React store code can trigger a React state change.
Output: One new context file + three modified store files with safe write paths.
</objective>

```typescript
import { createContext, useContext, useState, useEffect, type ReactNode } from 'react';

interface StorageErrorContextValue {
  permissionLost: boolean;
}

const StorageErrorContext = createContext<StorageErrorContextValue>({
  permissionLost: false,
});

// Module-level setter — stores call this (no React hook needed in store code)
let _reportPermissionLost: (() => void) | null = null;

export function reportPermissionLost() {
  _reportPermissionLost?.();
}

export function StorageErrorProvider({ children }: { children: ReactNode }) {
  const [permissionLost, setPermissionLost] = useState(false);

  useEffect(() => {
    _reportPermissionLost = () => setPermissionLost(true);
    return () => {
      _reportPermissionLost = null;
    };
  }, []);

  return (
    <StorageErrorContext.Provider value={{ permissionLost }}>
      {children}
    </StorageErrorContext.Provider>
  );
}

export function useStorageError() {
  return useContext(StorageErrorContext);
}
```

Then wrap EVERY `storage.write(...)` call in all three stores with the same try/catch pattern. The pattern is:

```typescript
try {
  await storage.write<T>('key', value);
} catch (e) {
  if (e instanceof DOMException && e.name === 'NotAllowedError') {
    reportPermissionLost();
    return;
  }
  throw e;
}
```

CRITICAL: Do NOT re-throw `NotAllowedError` — just call `reportPermissionLost()` and `return`. All other errors ARE re-thrown.
