**Status logic (compute inside component, not a separate file):**
```typescript
type Status = 'at-target' | 'near-target' | 'below-target';

function getStatus(account: Account): Status {
  if (account.targetCents === 0) return 'at-target'; // no target = always "ok"
  if (account.balanceCents >= account.targetCents) return 'at-target';
  if (account.balanceCents >= account.targetCents * 0.8) return 'near-target';
  return 'below-target';
}
```

**Status dot colors (Tailwind classes):**
- `at-target`: `bg-green-500`
- `near-target`: `bg-yellow-400`
- `below-target`: `bg-red-500`

**Inline editing behavior:**
- Default: show balance as `formatCents(account.balanceCents as Cents)` text (not an input)
- Click on the balance text: switch to an `<input type="text">` pre-filled with the raw number (e.g. "1500.00" — formatCents / 100, two decimal places)
- `onKeyDown` handler: Enter → `parseCents(inputValue)` → call `onBalanceChange(account.id, newCents)` → exit edit mode. Escape → exit edit mode without saving.
- `onBlur` on the input → save (same as Enter)
- While editing, show a small hint: `"Press Enter to save, Esc to cancel"`

**Card layout (using shadcn Card components):**
```
[status dot] Account Name                    [role badge, muted]
             €1,234.56  ← clickable balance
             Target: €3,000.00               (omit if targetCents=0)
```
Use `Card`, `CardContent` from shadcn. Keep it compact — single card height ~100px. No unnecessary padding.

**State:** Use React `useState` for `editing: boolean` and `editValue: string`.

Import: `Account` from `@/types/domain`, `formatCents`, `parseCents` from `@/lib/cents`, `cn` from `@/lib/utils`. Cast all cents values with `as Cents` when passing to `formatCents` or `parseCents`.
